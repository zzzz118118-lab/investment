# 제닉 트래커

제닉(123330) 투자 판단용 추적 도구. 마스크팩 수출 통계를 매월 수집해 누적한다.
(2026년 하나·교보·유안타 제닉 리포트 3건의 핵심 변수를 추적할 목적)

soil-tracker와 같은 계열이고 `charts.py` · `store.py`는 그대로 가져다 쓴다.
차이는 헤드라인 지표다. S-Oil이 정제마진(일별)이라면 제닉은
**마스크팩 수출액(월별)** 이 선행지표 자리에 온다.

## 배포

**https://zzzz118118-lab.github.io/investment/jenik/**

S-Oil 트래커와 같은 저장소(`investment`)·같은 Pages 사이트를 쓴다. 루트가 S-Oil,
`/jenik/`이 여기다. 두 페이지는 헤더에서 서로 링크된다.

저장소 루트의 `.github/workflows/daily.yml` 하나가 둘 다 빌드한다. 제닉 단계는
`continue-on-error`라 **제닉 수집이 실패해도 S-Oil 배포를 막지 않는다.**
`site/`는 gitignore 대상이고 CI가 매 실행마다 새로 만든다.

수출 통계는 월 1회(관세청 15일경) 갱신이라 매일 돌 필요는 없지만, S-Oil이 매일
도는 김에 같이 돈다. 주가와 아마존/검색량 수기 데이터가 있어 손해는 없다.

## 실행

```bash
pip install -r requirements.txt

python run_daily.py            # 수집 + 대시보드 한 번에
python dashboard.py --open     # 대시보드만 다시 만들고 브라우저로 열기
```

`site/index.html` — 의존성 없는 단일 HTML(외부 로드 없음). 그냥 열면 된다.

## 왜 마스크팩 수출인가

제닉은 하이드로겔 마스크팩 ODM이다. 자체 브랜드가 아니라 고객사 수출 물량을
만들기 때문에 **회사 매출이 국가 수출 통계를 후행**한다. 세 리포트 모두 이
수출 증가를 실적 전망의 1차 근거로 든다.

| 리포트 | 날짜 | 요지 |
|---|---|---|
| 하나증권 박종대 | 2026-06-05 | 2Q26 매출 396억(YoY +61%), OPM 19.2% 전망. 12MF PER 7.1배 저평가 |
| 교보증권 권우정 | 2026-06-24 | 탐방노트. 라인 8→11호기(+38%), 가동률 여전히 100% 초과. 26F PER 6배 |
| 유안타증권 | 2026-05-14 | 수출 확대 + CAPA 확대. 추정치는 미제시 |

## 카드 순서 바꾸기

각 카드 우상단의 손잡이(⠿)를 잡아 끌면 순서가 바뀐다. 키보드로도 되는데,
손잡이에 포커스를 준 뒤 ↑↓ 를 누르면 된다(드래그만 되면 접근성이 없다).

순서는 **브라우저 localStorage**(`jenik.cardOrder.v1`)에 저장된다. 서버가 아니라
브라우저에 두는 이유는 페이지를 매일 CI가 새로 만들기 때문이다. 그래서
기기·브라우저마다 순서가 따로 논다. 되돌리려면 헤더의 '카드 순서 초기화'를 누른다.

`data-card` 키는 카드 내용이 바뀌어도 그대로라 저장된 순서가 유지된다.
나중에 카드를 **추가**하면 저장된 목록에 없으므로 맨 뒤에 붙고, **제거**하면
저장값에 남은 키는 무시된다. 어느 쪽이든 카드가 사라지지 않는 것을 확인했다.

드래그는 손잡이를 누른 동안에만 켜진다(`el.draggable`). 카드 전체를 draggable로
두면 차트의 크로스헤어·툴팁 mousemove와 텍스트 선택이 막힌다.

## 무엇이 자동이고 무엇이 아닌가

매일 07:00 KST에 GitHub Actions가 도는데, 소스마다 자동화 가능 여부가 다르다.
대시보드 맨 아래 '데이터 갱신 상태' 카드와 `site/status.json`이 현재 상태를 보여준다.

| 소스 | 자동 | 주기 | 비고 |
|---|---|---|---|
| 마스크팩 수출 (관세청 API) | ✅ | 월 1회 | 매월 15일경 전월 확정 |
| 주가·시총 (네이버) | ✅ | 매일 | |
| 실적·컨센서스 (네이버) | ✅ | 매일 | `financials.py`가 누적 저장 |
| 아마존 순위 | ❌ | 수동 | ToS·CAPTCHA. 아래 참조 |
| 바이오던스 검색량 | ❌ | 수동 | Exploding Topics 유료 API 필요 |
| 리포트 추정치 | ❌ | 수동 | 새 리포트 나올 때 |

자동화 못 하는 건 **갱신 시점 알림까지를 자동화**했다. 기대 주기의 2배를 넘기면
대시보드에 경고 배너가 뜨고 Actions 실행 로그에도 `** 갱신 필요:` 줄이 남는다
(`freshness.py`).

### 실적이 자동으로 늘어난다

네이버는 최근 3개년·6분기만 보여주고 시간이 가면 창이 밀린다. `financials.py`가
매 실행마다 `data/financials.csv`에 쌓으므로 **과거 분기를 잃지 않고**, 새 분기가
발표되면 동행 차트가 저절로 늘어난다. 확정 실적은 덮어쓰지 않는다.

`config.QUARTERLY_REVENUE`(교보 도표1)는 CSV가 닿지 않는 1Q24~4Q24를 채우는
용도로만 남아 있다. 겹치는 구간은 네이버 값과 일치하는 것을 확인했다.

### 컨센서스 변화 추적

추정치가 바뀌면 `data/consensus_history.csv`에 (날짜, 기간, 항목, 이전, 이후)를
한 줄 남긴다. 값보다 **상향/하향 자체가 신호**라서 이력을 따로 둔다.

## 검증과 함정

### 리포트의 "마스크팩 수출액"은 세부코드가 아니라 HS 330790 전체다

이게 이 프로젝트에서 가장 중요한 확인 사항이다. 두 계열의 수준이 상당히 다르다.

| 연도 | 마스크팩 단품 (3307904000) | HS 330790 전체 | 유안타 리포트 |
|---|---|---|---|
| 2022 | 4.53억$ | **6.12억$** | 6.12억$ |
| 2023 | 3.71 | **5.38** | 5.38 |
| 2024 | 4.81 | **6.99** | 6.99 |
| 2025 | 6.08 | **8.73** | 8.73 |

유안타의 "1Q26 수출 2.49억달러(YoY +37.2%)"도 전체 합계와 소수점까지 일치한다.
**리포트 숫자와 대조할 때는 반드시 `total_usd`를 쓸 것.** 대시보드는 헤드라인에
단품(제닉의 실제 대응 품목)을, 비교용으로 전체를 함께 싣는다.

### 하나증권의 59% / 70%는 재현되지 않았다

하나 리포트의 "1분기 마스크팩 수출 YoY 59% 증가, 2분기 70% 상회"는 두 계열
어느 쪽과도 맞지 않는다(단품 +29%, 전체 +37%). 참고로 HS 3307909000 '기타'가
1Q +58% / 2Q +89%로 근접하지만 근거가 없어 추정에 그친다. 다른 기준의 통계로
보이며 **확인되지 않았다**. 대시보드 각주에 이 불일치를 그대로 남겨 두었다.

### API 제약 (soil-tracker/customs.py와 동일)

1. `hsSgn`은 6자리까지만 먹는다. 10자리를 넣으면 빈 결과가 온다.
   330790으로 조회한 뒤 `hsCode`로 갈라낸다.
2. 조회 구간이 길면 빈 결과가 온다. 연 단위로 끊어 요청한다.
3. **실패해도 예외가 아니라 빈 XML이 온다.** 행 수를 반드시 확인할 것.

### 3307904000은 2019년 신설

그 이전에는 마스크팩 단품 집계가 없다(2015~2018 `mask_usd` = 0). 장기 추이를
볼 때는 `total_usd`를 써야 한다. 대시보드의 '전체' 뷰가 2019년부터인 이유다.

### 갱신 주기

관세청은 매월 15일경 전월 확정치를 반영한다. 2개월 이상 밀리면 대시보드
상단에 경고 배너가 뜬다. 조용히 낡은 숫자를 보는 상황을 막는 장치다.

### 진행 중 분기 제외

분기 YoY는 3개월이 다 찬 분기만 계산한다. 달이 모자란 분기를 그대로 쓰면
증가율이 크게 왜곡되기 때문이다.

## 아마존 Best Sellers 순위 (수동 스냅샷)

교보 탐방노트가 짚은 지표 — "최대 고객사가 아마존 뷰티 TOP 10 내 꾸준히 랭크,
마스크팩 카테고리 1위".

### 갱신 방법

```bash
# 1. 브라우저로 아래 페이지를 연다
#    https://www.amazon.com/gp/bestsellers/beauty/11061121
# 2. 콘솔에 capture.js 내용을 붙여넣고 실행, 출력 JSON을 저장
# 3. 적립
python amazon.py --ingest data/snapshots/20260719_masks.json
python dashboard.py --open
```

스냅샷이 2장 이상 쌓이면 대시보드에 순위 추이 차트가 나타난다(y축 반전 — 위가 1위).

### 왜 수동인가

아마존은 ToS에서 스크래핑을 금지하고, 자동 반복 요청은 CAPTCHA로 막힌다.
우회는 만들지 않는다. 조용히 깨지는 파이프라인보다 사람이 한 번 여는 쪽이 낫고,
수출 통계가 월 1회 갱신이라 주기도 어차피 맞는다.

자동화가 꼭 필요하면 Keepa API(유료, BSR 시계열 제공)나 아마존 공식
PA-API(Associates 계정 필요)가 합법적인 경로다.

### 카테고리 노드 고르기

계층은 이렇게 내려간다:

```
11060451 피부 관리
└ 11060711 페이스
  └ 11062031 트리트먼트 및 마스크
    └ 11061121 마스크        ← 이걸 쓴다
```

처음엔 `11062031`을 썼는데 **TOP 20 중 실제 마스크가 3개뿐**이고 나머지는
세럼·토너·여드름 패치였다. 하위에 마스크 전용 노드가 있는 걸 뒤늦게 발견해
2026-07-19에 `11061121`로 갈아탔다. 바꾸고 나니 20개 중 17개가 마스크고,
BIODANCE가 **#2 → #1**로 올라온다(같은 날, 같은 시각 기준).

노드를 바꾸면 **순위 체계가 달라 이어 붙이면 안 된다.** CSV에는 node 컬럼이
있고 `amazon.for_node()`가 현재 노드만 걸러낸다. 과거 노드 스냅샷은 지우지 않고
남겨 두되 대시보드에는 그리지 않는다.

처음 받았던 검색 URL(`/s?k=mask+pack`)은 **쓰지 않는다.** 판매 순위가 아니라
관련도 정렬이고 상위 4개가 후원 광고이며, 방역용 마스크가 섞여 들어온다.

### 캡처 시 주의

- **평점은 마지막 숫자를 써야 한다.** `별 5개 중 4.6`에서 첫 숫자를 잡으면
  전부 5.0이 된다. 처음에 이렇게 틀렸다.
- 카테고리 페이지를 이동한 뒤 스니펫을 돌려야 한다. 상위 카테고리에서 실행하면
  다른 순위표가 잡힌다(이것도 한 번 겪었다).
- 항목이 20개 미만이면 셀렉터가 깨진 것이다. `amazon.py`가 경고를 낸다.
- 세션 로케일이 한국이라 **제품명이 한글로** 들어온다. 순위와 ASIN은 amazon.com
  기준으로 동일하지만, 배송처 설정이 노출 품목을 거를 여지는 확인하지 않았다.

### 추적 대상 ASIN

`config.WATCH_ASINS`. 현재 `B0B2RM68G2` = BIODANCE 바이오 콜라겐 리얼 딥 마스크
(하이드로겔 오버나이트, 2026-07-19 기준 이 카테고리 **#1**).

> **제닉이 이 제품을 만드는지는 확인되지 않았다.** 하나 리포트는 최대 고객사를
> 'H사'로만 표기한다. 하이드로겔 오버나이트 마스크라는 제품 성격이 제닉 주력과
> 일치해 후보로 올려둔 것이다. 확인 전까지 근거로 쓰지 말 것.

## 바이오던스 검색량 (Exploding Topics)

```bash
python searches.py --ingest "C:/Users/sungh/Downloads/biodance-xxxx.csv"
```

`data/searches.csv` — date, searches, is_forecast.

### 예측치가 섞여 있다

내보내기 파일은 **미래 12개월치 예측을 실측과 같은 칸에** 담아 보낸다.
메타의 `Volume`이 '현재 월 검색량'이라, 그 값과 일치하는 마지막 행이 실측
경계다. `searches.py`가 이 규칙으로 `is_forecast`를 세우고 대시보드는 실측만
그린다. 2026-07-19 받은 파일은 **2026-06이 경계**였고 예측이 12개월 붙어 있었다.

### 토픽 오염 가능성

CSV의 Description은 1960년대 롤란도 토로의 무용치료 'Biodance'를 설명한다.
K뷰티 브랜드가 아니다. 다만 2023년 9월 이전 검색량이 0이고 그 뒤 폭증하는
형태라 실제 시계열은 브랜드 검색으로 보인다. 동음이의 오염은 남아 있다.

### 동행 차트 — 왜 지수인가

검색량(건수)·수출(백만달러)·매출(억원)은 단위가 전혀 달라 한 축에 못 얹는다.
**1Q24를 100으로 지수화**하고, 매출이 분기 단위라 셋 다 분기로 맞춘다.
부분 분기는 제외한다.

읽을 때 주의: 이 차트는 **인과가 아니라 시기적 동행**만 보여준다.
바이오던스와 제닉의 공급 관계는 확인되지 않았다.
