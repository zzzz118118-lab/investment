# S-Oil 트래커

정제마진·크랙 스프레드·유가를 매일 수집해 누적하는 파이프라인.
(2026-07 신영증권·하나증권 S-Oil 리포트의 핵심 변수를 추적할 목적)

## 실행

```bash
pip install -r requirements.txt

python backfill_petronet.py 2006   # 최초 1회: 20년치 백필 (약 1분)
python backfill_hana.py            # 하나증권 참조표 병합 (OSP 등)
python collect.py                  # 이후 매일
python dashboard.py --open         # 대시보드 생성 후 브라우저로 열기
```

`data/dashboard.html` — 헤드라인 마진, 지표 타일, 정제마진 추이(**일/주/월 전환**),
크랙 5종 120영업일, 연평균 20년. 각 차트마다 표 뷰가 붙어 있다.
의존성 없는 단일 HTML이라 그냥 열면 된다(외부 로드 없음, 44KB).

## 데이터 출처

한국석유공사 페트로넷. 인증 불필요.

- **메인 페이지 차트** (`main2.jsp`) — Chart.js 데이터셋에 22영업일 시계열이 박혀 있다. 시딩용. 원유 3종 + 제품 3종만.
- **상세 테이블** (`/v4/sub.jsp` POST) — 최근 7영업일, 제품 8종 + 원유 4종. 매일 갱신용. 갱신주기는 화~토(전일자 반영).

## 알려진 제약

1. **연 단위로 쪼개야 백필된다.** 서버에 행 수 상한이 있어 1.5년(약 390행)까지는 되고 6년은 빈 결과가 온다. `backfill_petronet.py`가 연 단위로 나눠 요청한다. 20년 = 약 1분.
   - 기간 조회가 먹으려면 네 가지가 모두 맞아야 한다. 하나라도 틀리면 **에러가 아니라 빈 결과**가 와서 원인을 찾기 어렵다:
     1. 제품코드 구분자가 `\,\'` — **백슬래시 콤마**. 그냥 `,`면 실패
     2. `:DISP1='...'`를 Parameter 끝에 붙일 것
     3. 메뉴 POST로 서브페이지를 한 번 로드해 **세션을 예열**한 뒤 조회할 것
     4. 기간 2일 이상이면 `InitialLoadFile`을 `_EXT` 템플릿으로 바꿀 것
2. **정제마진은 자체 산출이지만 검증됨.** 페트로넷이 게시하지 않아 계산하되, 수율을 하나증권 공표 시계열에 역산 적합해 사실상 그대로 재현한다(아래 검증 참조). 하나증권이 산식을 바꾸면 어긋날 수 있으므로 표를 받을 때마다 `hana_reference.csv`에 추가하고 `fit_yields.py`를 재실행할 것.
3. **기유(base oil) 미포함.** 무료 실시간 소스가 없다. 관세청 수출단가(월간) + 뉴스 모니터링으로 별도 구현 예정.

## 검증 (2026-07-18)

하나증권 '화학 제품가격 동향'표(2026-07-15 기준)와 대조. 원시 가격은 같은
페트로넷 소스라 **완전히 일치**했고, 크랙은 유종 등급을 맞추자 일치했다.

| 항목 | 하나 7/15 | 자체 7/15 | 하나 7/14 | 자체 7/14 |
|---|---|---|---|---|
| Dubai | 79.3 | 79.29 | 83.2 | 83.20 |
| WTI | 79.6 | 79.60 | 79.3 | 79.34 |
| 납사 Crack | 11.3 | 11.31 | 7.3 | 7.34 |
| 가솔린 Crack | 30.5 | 30.52 | 28.3 | 28.28 |
| 등유 Crack | 68.8 | 68.84 | 74.8 | 74.82 |
| 경유 Crack | 66.5 | 66.50 | 69.8 | 69.81 |
| B-C Crack | 5.6 | 5.59 | 0.1 | 0.13 |
| Complex Crack | 43.5 | 43.51 | 44.4 | 44.40 |

나프타 $/bbl → $/Ton 환산계수는 약 **9.21** (하나증권 표는 $/Ton 표기).

### 정제마진 수율 역산

Complex/Simple Crack은 처음엔 4~5$ 어긋났고 잔차가 일정하지 않아 상수 보정이
통하지 않았다. 하나증권 표 11일치(07-01~07-15)를 모아 크랙 5종에 최소자승
적합하니 계수가 복원됐다 (`fit_yields.py`).

| 유종 | Complex 수율 | Simple 수율 |
|---|---|---|
| 납사 | 0.165 | 0.095 |
| 가솔린(95RON) | 0.163 | 0.098 |
| 등유 | 0.160 | 0.129 |
| 경유(0.05%) | **0.384** | 0.225 |
| B-C(180cst) | 0.059 | **0.424** |
| 합계 | 0.931 | 0.971 |
| 운영비 | 0.20 | −0.03 |

- **Complex R² = 1.0000** (잔차 최대 0.04), **Simple R² = 0.9999** (최대 0.05)
- 원표가 소수 첫째자리 반올림이므로 사실상 완전 재현
- 수율 합이 1보다 작은 만큼(6.9% / 2.9%)이 자가소비·손실. 물리적으로 타당하다
- Complex는 경유 38%로 고도화 비중이 높고, Simple은 B-C 42%로 잔사유가 그대로 남는다 — 두 산식의 성격과 일치

### 참조표 병합

`python backfill_hana.py` — `data/hana_reference.csv`의 크랙·마진·OSP를
`prices.csv`에 병합한다. 페트로넷 실측이 있는 칸은 덮어쓰지 않되, 마진과 OSP는
하나증권 공표값을 정본으로 쓴다. 새 표를 받으면 CSV에 행을 추가하고 재실행할 것.

## 코드 주의사항

- 원유 테이블 컬럼 순서는 `Dubai, Brent, WTI, Oman`. 메인 차트는 `Dubai, WTI, Brent`. **순서가 다르다.**
- 테이블 파싱 시 값 정규식은 소수점을 **반드시** 요구해야 한다. 정수를 허용하면 다음 행 `07월`의 `07`을 값으로 먹어 행이 격일로 누락된다.

## 집계 단위

`aggregate.py` — 일별에서 주/월/연 평균을 낸다. 부분 구간 왜곡을 막기 위해
최소 관측 수(주 3, 월 10, 연 100)를 채운 구간만 집계한다.

주간·연간은 하나증권 표와 정확히 일치한다 (주간 7-2nd 34.3 / 7-1st 32.2,
연간 '25 10.0 / '24 8.9). **월간은 1 내외로 어긋난다** — 결측일 처리 방식이
다른 것으로 보이며 원인은 미확인.
